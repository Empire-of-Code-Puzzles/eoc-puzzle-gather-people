私たちは緊急事態に備える必要があります、そこで避難過程のモデルを開発することにしました。 建物の部屋は様々な長さの廊下で繋がれています(しかしそれらは孤立しているかもしれません)。 それぞれの部屋はいつでも様々な数の居住者を含む可能性があります。 私たちは与えられた時間でどれだけの人々が避難場所にたどり着けるか計算しなくてはいけません。

部屋と廊下の情報は行列 (M) で表現されます。 それぞれの部屋は0からNの番号を持っていて、行列の大きさはNxN です。 与えられた情報の対角成分はどれだけの人々が部屋にいるかを表しています -- M[i][i]は i番目の部屋に居る人々の数です。
行列の他のマスは廊下についての情報が与えられ、具体的には部屋i から部屋j へ移動するには どれだけの時間(分)が必要か(M[i][j] == M[j][i]) を表しています。 もし時間が0ならそれらの部屋をつなぐ廊下は存在していないということです。 避難場所は番号0 の部屋です。

あなたは建物の行列と避難時間を分で与えられます。 人々は建物を避難するのにいつも最短経路を使います。 指定された時間で、どれだけの人々が逃れられますか？ このミッションでは部屋を通り抜けるのに時間はかからず、廊下を移動する時のみ時間がかかります。 避難にかかる合計時間が与えられた時間以下の部屋は避難すると考えます。

例えばこの画像を見てください 

[[ 0, 40,  0, 40,  0,   0],
 [40,  6,  0,  0, 40,   0],
 [ 0,  0,  3,  0, 15,   0]
 [40,  0,  0,  4, 40,  15]
 [ 0, 40, 15, 40,  1,   0]
 [ 0,  0,  0, 15,  0,   2]]

もし与えられた時間が80ならば1, 3, 4, 5の部屋の人々は完全に避難可能で結果は6+4+1+2=13 となります。

私たちはこのチャレンジにもう一つのルールを課します。 これはコードゴルフミッションであなたの主なゴールはあなたのコードを出来る限り短くすることです。 コードを短くすればより多くのポイントを獲得することができます。 あなたのスコアは動的でコードの長さに直接関連しています。 参考までにスコアは使われた文字の数に基づいています。 250 文字が許容される最大文字数でこの時は0点です。 250文字から1文字減るごとに1ポイント獲得することができます。 例えば150文字の長さのコードでは100ポイント獲得することになります。 このミッションでは空白はカウントしますがインデントはカウントしません。

入力 2つの引数。建物に関する情報が整数のリストのリストとして与えられ、避難時間が整数として与えられます。

出力 時間内に避難した人々の数を整数として出力します。

1 < len(building) <= 10
all(building[i][j] == building[j][i] for i in range(len(building)) for j in range(len(building))))
building[0][0] == 0
